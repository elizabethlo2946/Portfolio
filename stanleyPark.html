Stanley Park

a first nation's perspective. android application.

Role
-

  • 1 back-end development
  • 2 user flow
  • 3 user interface

Goal
-

To create a fully functional application using at least: shared preferences (eg. saving log-in information), a local SQLite database (eg. saving user text-input), implicit and explicit intents, at least one sensor, and camera functionality

Tools
-

  • Android Studio
  • Illustrator

Project Brief
-

November 2017
Course - IAT 359 introduction to mobile computing

An informational app on Stanley Park, designed for locals and tourists who want to know the meaning behind particular landmarks or art pieces from a First Nation's perspective.

The current City of Vancouver Website (as of Fall 2017) only explains what each landmark and art piece is, but does not explain why it is important to First Nations People.

Context
-

Over 4 months, I explored the basic functions of Android Studio such as intents, sensor access, saving user information, and implementation of Google Maps. It was an individual project, and I was able to create my app in a month.

Audience
-

This app was created as a prototype for locals and tourists who want to know more about the importance of Stanley Park landmarks to the First Nations people.

Each piece shows the description provided by the City of Vancouver's website, and below, ideally First Nations people are able to log in and input additional meanings or stories into the database. When users press "show more", that information will appear.

Role
-

I planned most of the app through site maps and listing out the concepts in order to carry out the functionality to link to other events. The structure of the site map also helped me understand the basic user flow. I planned out my UI through Illustrator to help visualize the basic user experience of how people are taken through the app.

Take-aways
-

I was able to learn basic Android functionality of an app and how to interact with different screens and other apps. The theoretical part of Android is quite simple, but the actual implementation is quite difficult using Android Studio (as with most code).
