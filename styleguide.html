Project 1: LAMPS

Responsive web design

Team

  • Abel Abraham

A fictional lighting company’s website, created to help solidify concepts of building an accessible website across multiple devices and browsers.

Context

This is my first time coding a website that implements design and functionality together. Within 4 weeks, my partner and I were able to go through an initial style guide, a site map, and an additive process of iteration to completion.

Role

I created the visual style of the website, starting with the home page menu, then the grid structure for the rest of the page. I also designed the cart, about, rating pages and the footer, which follow grid structures that can be found on the home page. This includes responsiveness and functionality of all buttons, hover states, and JavaScript of the collapsible menu.

I also did the style guide to hold all of the visual elements for the website and a quick citations page. You can check the Github for more specific details on contributions.

take-aways

Naming conventions are useful for those who did not create the document. Although I created most of the class and ID names, they could have been named more clearly for my partner to iterate and work with as he started his pages after me and had to follow my naming conventions. I did not receive any complaints about naming, but I think that it would have been a much quicker process for my partner if it was more intuitive.

Project 2: Android application development

Project brief

An informational app on Stanley Park, designed for locals and tourists who want to know the meanings behind particular landmarks or art pieces from a First Nations perspective.

The current City of Vancouver Website (as of Fall 2017) only explains what each landmark and art piece is, but does not explain why it is important to First Nations People.

context

Over the semester, we explored use cases of Android Studio such as intents, sensor access, saving user information, and implementation of Google Maps. As this was an individual project, I was able to create my app in a month.

role

Most of my planning was done through site maps and listing out the concepts in order to carry out the functionality to link to other events.
I planned out my UI through Illustrator to help visualize the basic user experience of how people are taken through the app

take-aways

I was able to learn basic Android functionality of an app and how to interact with different screens and other apps. The theoretical part of Android is quite simple, but the actual implementation is quite difficult using Android Studio (for me, anyways).

difficulties

  • Learning Android Studio’s syntax was difficult as a beginner, even with previous Java experience. Because it has so many built in strings that perform legitimate tasks, I had to resort to searching a lot of basic questions online. Without the help of existing code from class, it would have taken me much longer to implement all functionality such as the Recycler and Card View, and the user log-in details.
  • After the 2nd milestone of implementing all functionality of the app, I decided to relocate my image folder and remove some previous images. My app still worked, but no images would show, and undoing the relocation did not help the cause. I ended up redoing the entire process, by using a Card View tutorial online for the home page and salvaging code from my 2nd milestone. That was not a delightful experience.
